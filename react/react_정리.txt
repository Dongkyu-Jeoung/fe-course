1. 리액트의 탄생
    (1) 탄생 배경
        - React는 페이스북의 개발팀이 화면 업데이트를 위해 개발하던 중 탄생
        - React는 자바스크립트 기반의 라이브러리
        - 2013년도 등장한 선언형 방식으로 UI를 표현
        - 현재는 2024 (React.19v)을 API 지원 : 항상 최신 버전으로 개발 권장

    (2) 왜 새로운 기술이 필요했나?
        - 기존의 기술 한계점 : 정적인 웹 구축에 초점 -> 대규모 앱 구축에 부적합
        - 페이스북의 요청 : 
            다양한 인터랙션(사용자와의 양방향 소통)
            변화가 잦은 대규모 앱 필요 

    React 핵심 목적 - 변화가 자주 일어나는 대규모 어플리케이션에 적합한 쉽고 효율적인 라이브러리 개발

2. 리액트의 특징 3가지
    (1) 컴포넌트 기반
        - UI를 독립적이고 재사용 가능한 컴포넌트 단위로 분리
    (2) 가상돔(Virtual DOM)
        - 실제 DOM을 복제하여 가상돔에서 필요한 부분만 업데이트 진행 후 한번에 실제 DOM에 반영 (빠른 업데이트)
    (3) 단방향 데이터 흐름
        - 부모 컴포넌트 -> 자식 컴포넌트로 데이터를 전달
        - 자식 -> 부모로는 안됨

3. 리액트 앱 생성 - 리액트 프로젝트
    (1) Vite(비트) - 리액트 라이브러리를 제공하는 오픈 API 
    (2) 기존 CRA(Create React App)를 대체하는 라이브러리
    (3) Vite를 활용한 리액트 앱 생성
        1) 개발 환경 구성 - Node.js 설치 (node -v / npm -v)
        2) 리액트 앱 생성 : my-app 
            형식 : npm create vite@latest [앱이름]
            예) npm create vite@latest my-app       

4. 리액트 앱 동작 순서 
    (1) npm run dev         // 서버 실행 http://localhost:5173/index.html
    (2) index.html  렌더링
    (3) main.jsx  호출
        - index.html의 root 아이디에 import한 컴포넌트 추가
    (4) App 컴포넌트 렌더링 : virtual DOM

5. 컴포넌트
    (1) 정의
        - UI를 구성하는 독립적이고 재사용 가능한 단위
        - 함수가 HTML을 반환하는 구조 : return()
        - 레고 블록을 조합하여 페이지를 구성
    (2) 종류
        - 함수형 컴포넌트 : 2018년 이후 사용됨, 현재 표준
            function [컴포넌트명] () {
                return ();
            }
        - 클래스형 컴포넌트 : 거의 사용 안함
            class [클래스명] extends React.Component {
                render();
            }
    (3) 특징
        - 재사용성
        - 독립성 : 파일이름 하나당 컴포넌트 하나 CSS 하나
        - 조합 가능 : 컴포넌트를 조합하여 화면을 구성
    (4) 구성요소
        형식 : function [컴포넌트명] ( props ) {
                // 로직, 컴포넌트 상태관리 (React Hooks)
                return();
            }

            const [컴포넌트명] ( Props ) => {
                // 로직, 컴포넌트 상태관리 (React Hooks)
                return();
            }

        - 컴포넌트 명은 PascalCase (첫글자 대문자)
        - 컴포넌트의 인자값은 props로 전송되며, 단방향으로 진행
        - return 함수를 통해 화명 렌더링(출력)
        - return 함수는 JSX 문법이 적용됨

    (5) 부모 컴포넌트 vs 자식 컴포넌트
        - 부모 컴포넌트 : index.html에 실제 렌더링 되는 컴포넌트
        - 자식 컴포넌트 : 부모 컴포넌트의 return 함수에서 호출되는 컴포넌트

    (6) Export / Import
        - 컴포넌트 앞에 export를 추가
        export (default) function [컴포넌트명] () {
            return()
        }
        - 별칭(Alias)을 이용하여 자식 컴포넌트를 import 함
        - export default 컴포넌트는 컴포넌트명 그대로 사용
        - export 컴포넌트는 {컴포넌트명} 형식
        import [별칭] from '컴포넌트위치.확장자';

6. JSX (JavaScript Extension)
    (1) JSX 란?
        - 자바스크립트 + HTML 태그를 포함한 리액트 확장 문법
        - 공식 JS 문법은 아니지만 리액트 개발팀에서 적극 권장
        - 중괄호 {}를 사용하여 HTML안에 JS 표현식 사용가능

    (2) JSX에서 사용 가능한 표현식
        - 산술 표현식 : {numA + numB}
        - 문자열 표현식 : {strA + strB + '홍길동'}
        - 논리 표현식 : {booleanA || booleanB}
        - 객체 표현식 : Object Literal은 객체명.프로퍼티명으로 호출 / 객체명으로는 접근 불가

    (3) JSX 문법
        - 모든 태그는 반드시 closing(/>) 태그를 가져야함
        - 최상위 태그를 반드시 하나만 가지고 있어야함
        - 최상위 태그를 <></> 빈태그로 가질 수 있음

    (4) 조건부 렌더링 - 삼항연산자, if 조건문
        
    (5) JSX 스타일링
        - 인라인 스타일링 : CSS 속성을 객체로 작성, camelCase사용
        ex) <h1 style={{backgroundColor:'red', ...}}> Hello React </h1>

        - CSS 파일로 작성하고, 컴포넌트에서 import 
        ex) import 'CSS파일위치';

        클래스 적용시 class="" 이거 안씀
        <h1 className="body">CSS 클래스명 적용</h1>

7. 컴포넌트에 값 전달하기 : 단방향 (부모 -> 자식)
    (1) Props란? Properties 약자
        - 부모 컴포넌트가 자식 컴포넌트에게 전달하는 단일 객체 

    (2) Props 전달 방식
        1) 기본적인 전달방식
            function Parent() {
                return (
                    <Child name="홍길동" />
                )
            }
            function Child(props) {        // props = {name : '홍길동'}
                return (
                    <h1>Hello~ My name is {props.name}</h1>
                )
            }

        2) 구조분해 할당
            function Parent() {
                return (
                    <Child name="홍길동" age={30} />
                )
            }
            function Child({name, age}) {        
                return (
                    <h1>Hello~ My name is {name}</h1>
                    <p>I'm {age} years old</p>
                )
            }

        3) 스프레드 연산자
            function Parent() {
                let info = {
                    name = "홍길동",
                    age = {20}
                }
                return (
                    <Child {...info} />
                )
            }
            function Child({name, age}) {      // {name : '홍길동', age : 30}
                return (
                    <h1>Hello~ My name is {name}</h1>
                    <p>I'm {age} years old</p>
                )
            }

        4) Props로 컴포넌트 전달
            function PropsComp () {
                return (
                    <h1>PropsComponente!!</h1>
                )
            }
            function Parent() {
                return (
                    <Child>
                        <PropsComp/>
                    <Child/>
                )
            }
            function Child({ children }) {      
                return (
                    <h1>Hello~</h1>
                    {children}
                )
            }
8. 이벤트 처리하기
    (1) 이벤트란?
        - 웹사이트에 발생하는 사용자의 행위
        - 버튼 클릭, 텍스트 입력, 페이지 스크롤 ... 
        - 이벤트 핸들링 : 이벤트 발생 시 특정 코드가 처리되도록 만드는 작업
        - 이벤트 핸들러 : 이벤트를 처리하는 함수(객체)

    (2) HTML vs React 이벤트 핸들링
        - HTML : onclick(소문자) -> onclick="handleClick()"
        - React : onClick(camelCase) -> onClick={handleClick}
                                     -> onClick={ () => {} }   // 직접처리 시 콜백 함수 정의
        예) HTML
            <button type="button" onclick="handleClick()" />
            React                                    
            <button type="button" onClick={handleClick} />

    (3) 이벤트 객체 활용 - 이벤트 발생 시 호출되는 객체
        - 이벤트객체.target : 이벤트 객체 target 요소
        - 이벤트객체.target.name, 이벤트객체.target.innerText ...
        ex) 
            const handleClick = (e) => {            // 이벤트객체를 'e'로 받는거
                console.log(e.target)
            }

            <button type="button" name="버튼1" onClick={handleClick} />
            <button type="button" name="버튼2" onClick={ e => console.log(e.target) } />

    (4) 이벤트 객체를 활용하여 자식 -> 부모 값 전달 가능
        function 부모() {
            const handleParent = (4️⃣result) => {
                console.log('자식이 보내온 값 ->', result);
            }
            return (
                <자식 1️⃣click={handleParent}/>
            )
        }
        function 자식({ 2️⃣click }) {
            const handleChild = () => {
                // 부모에게 100 전달
                3️⃣click(100);
            }
            return (
                <button type="button" onClick={handleChild}>클릭하세요</button>
            )
        }

9. 상태관리(State) - useState
    (1) 컴포넌트가 관리하는 동적값이 변동되면, 리액트 컴포넌트가 자동으로 리렌더링
    (2) useState 사용법
        [형식] import { useState } from 'react'
            const [(동적값을 저장한)변수, set변수명] = useState(변수의 초기값);
            
            ex) Number :: const [number, setNumber] = useState(0);
                String :: const [string, setString] = useState('');
                Array :: const [array, setArray] = useState([]);
                Object :: const [obj, setObj] = useState({});

10. Ref - useRef
    (1) Ref란?
        - 실제 DOM 요소를 직접 참조하여 관리하는 기능을 가짐 ( = Reference)

    (2) useRef 사용법
        import { useRef } from 'react'
        const nameRef = useRef(null);
        function handleChange () {
            ...
        }
        <input type="text" name="name" ref={nameRef} onChange={handleChange} />

    (3) DOM 입력 값 가져오기 : nameRef.current.value
        DOM 입력 폼 포커스 설정 : nameRef.current.focus()

11. 리액트 라이프 사이클 & 리액트 개발자 도구 (react developer tools)
    (1) 라이프 사이클 3단계
        - Mount(마운트) : 처음 생성
        - Update(업데이트) : 변화, 업데이트
        - UnMount(언마운트) : 종료, 제거

    (2) Mount (마운트)
        - 컴포넌트가 처음으로 랜더링 되는 시점
        - 예) API 호출, 초기 데이터 랜더링 등..

    (3) Update(업데이트)
        - 컴포넌트가 리랜더링 되는 시점
        - 예) State 변경, props 변경, 부모 컴포넌트 변경 등

    (4) UnMount(제거)
        - 컴포넌트가 화면에서 사라질 때
        - 예) 이벤트 종료, 타이머 제거, 메모리 정리 등

    (5) useEffect로 라이프 사이클 제어
        import { useEffect } from 'react'
        
        export default function [컴포넌트명] () {
            useEffect( () => {
                // 1. 마운트 or 업데이트 시 출력
                console.log("Mount or Update!");

                // 2. 언마운트
                return () => {
                    console.log("UnMount!")
                }
            }, [ dependencies, ... ] );

            return (
                <></>
            )
        }

12. SPA - Single Page Application
    (1) 개념 : 한 개의 HTML 페이지에서 모든 화면의 전환이 이루어짐
                페이지 이동 시 새로고침 현상이 발생하지 않음
    (2) MPA(Multi Page Application)
        - 메뉴별 독립적인 페이지를 생성하여 화면의 전환이 이루어짐
        - 페이지 이동 시 새로고침 현상 발생
    (3) 라우팅(Routing)
        - 메뉴로 구성된 컴포넌트 페이지로 이동하는 방식
        - react-router-dom 패키지 : Link, NavLink 컴포넌트 포함..

13. 리액트 전역 객체 관리 방법
    (1) Context + Hooks(useContext, useReducer ...)
    (2) React-Redux + Reducer(useReducer)
    (3) React-Redux + Redux-Toolkit(@redux/toolkit) - store, slice

    (4) Redux-Toolkit(@redux/toolkit) 사용법
        1) store : 상점
        2) slice 생성(Reducer) : 상점에 진열되는 상품(데이터, 기능)
        3) useSelector : 진열된 store의 slice 상품 데이터 가져오기
        4) useDispatch : 진열된 store의 slice 상품 기능을 호출
        5) 패키지 설치
            npm i(install) react-redux @reduxjs/toolkit
        6) store, slice 생성 참고
            https://redux-toolkit.js.org/tutorials/quick-start
